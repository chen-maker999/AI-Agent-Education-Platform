# RAG 测试体系运行指南

## 测试体系概述

本测试体系包含三个层级，共 **200 个测试用例**：

- **单元测试 (Unit Tests)**: 156 个测试
- **集成测试 (Integration Tests)**: 44 个测试（含 18 个 Mock Embedding 评估测试）


## Mock Embedding 评估（新增）

### v1 vs v2 对比

| 版本 | 检索方式 | 数据泄露 | 速度 | F1 | 推荐使用场景 |
|------|---------|---------|------|----|------------|
| **v1** | 模拟检索（使用 expected_keywords） | ✅ 有 | < 0.3 秒 | 0.5715 | 代码逻辑验证、CI/CD |
| **v2** | 真实 BM25 检索（无数据泄露） | ❌ 无 | < 0.5 秒 | 0.6225 | 检索算法测试、真实评估 |

**核心发现**: v2 的 F1 反而比 v1 高 8.9%，因为 v2 的文档内容更丰富、关键词覆盖率更高。

## 评估对比

| 评估模式 | 需要 RAG 服务 | 需要 GPU | 速度 | F1 基线 | 推荐使用场景 |
|----------|--------------|----------|------|---------|--------------|
| Mock Embedding v1 | ❌ | ❌ | < 0.3 秒 | 0.5715 | 代码逻辑验证、CI/CD |
| Mock Embedding v2 | ❌ | ❌ | < 0.5 秒 | 0.6225 | 检索算法测试、无泄露评估 |
| Mock LLM | ✅ | ❌ | ~30 秒 | 待测定 | 无 GPU 环境、集成测试 |
| 完整评估 | ✅ | ✅ | ~60 秒 | 0.1095 | 生产环境、最终验证 |

**重要说明**:
- Mock 评估（v1/v2）的 F1 显著高于真实 RAG 评估，主要原因是：
  1. 评估指标计算方式不同（规则基 vs 真实对比）
  2. 文档库规模不同（~200 模拟文档 vs 数千真实文档）
  3. 检索难度不同（中文 vs 跨语言）
- **唯一可信的生产基线**: 完整评估 F1=0.1095

## 常见问题

### Q: 测试失败怎么办？

A: 检查以下几点：
1. 虚拟环境是否正确激活
2. 依赖是否完整安装：`pip install -r requirements.txt`
3. 查看具体错误信息：`python -m pytest tests/ -v --tb=long`

### Q: 集成测试需要运行 RAG 服务吗？

A: 部分集成测试需要 RAG 服务运行在 `http://localhost:8000`。如果服务未运行，相关测试会跳过或降级测试。

### Q: 如何添加新测试？

A: 遵循现有测试结构：
1. 在 `tests/unit/` 或 `tests/integration/` 目录创建测试文件
2. 使用 `pytest` 和 `pytest-asyncio` 框架
3. 遵循命名规范：`test_*.py`
4. 使用 `@pytest.mark.asyncio` 标记异步测试
